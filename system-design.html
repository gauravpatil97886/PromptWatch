02

The request lifecycle

Eight steps from keystroke to dashboard. Steps 1–5 are on the critical path and add no latency; everything else happens after the user already has their answer.

STEP 01

Point at the gateway

The employee exports ANTHROPIC_BASE_URL=http://127.0.0.1:9099. Claude Code, the VS Code extension, and SDK apps all honor it.

STEP 02

Request hits the agent

The agent records who & where — device, hostname, OS, system user, client IP, user-agent — without touching the request body.

STEP 03

Forward, untouched

The request is streamed straight to api.anthropic.com. Headers, body, and the user's key pass through unmodified.

STEP 04

Stream back immediately

The response streams to the client in real time. The user notices nothing — zero added latency.

STEP 05

Analyze after the fact

Once the stream ends, SSE events are parsed, the text is extracted, and security.analyze() scores it on the raw content.

STEP 06

Redact, then persist

Detected secrets and PII are masked. The redacted record is written locally and shipped to the collector over HTTPS with a per-agent key.

STEP 07

Re-evaluate centrally

The collector re-runs detection + compliance on the raw text, re-masks as defense-in-depth, and stores the result encrypted, bound to the agent identity.

STEP 08

Surface & prove

The dashboard updates live with toast alerts; every admin action is hash-chained; one click produces a signed compliance evidence pack.

03

Two independent paths

The genius is the split: the path that keeps Claude working never depends on the path that does the monitoring.

⚡

The forward path

Employee → agent → Anthropic. Streaming preserved, request unmodified. This path is synchronous and minimal — its only job is to get the answer back instantly.

PROPERTY · zero added latency · key never logged
🛟

The telemetry path (fail-open)

Logging, detection, and shipping happen asynchronously. If the collector or network is down, the user's Claude call still succeeds — the event spools to disk and replays later.

PROPERTY · Promptward can never break Claude
If Promptward disappeared mid-request, nothing would break.
The forward is independent of the server. Monitoring is additive — never a dependency on the critical path.
04

Inside the backend

What actually runs when an event lands — the governance pipeline, layer by layer.

🎯

Threat detection

Credentials (HIGH), prompt injection & exfiltration intent (MEDIUM), oversized prompts (LOW). Runs on raw text, enforced server-side.

common/security.py
🩹

Redaction · defense-in-depth

Secrets and PII are masked at the agent and re-masked at the collector, so a raw payload never lands in the store.

common/redact.py
📋

Compliance & DLP

PII/PHI/PCI detectors (Luhn-checked cards), configurable AUP policy packs, and model allow/deny governance feed a violation register.

common/compliance.py
⛓️

Tamper-evident audit

Every admin action and finding is appended to a SHA-256 hash chain; verify() detects any insertion, edit, or deletion.

server/audit.py
📜

Signed evidence packs

One command yields an HMAC-signed, framework-mapped artifact (EU AI Act · NIST · ISO 42001 · SOC 2 · GDPR), as JSON or print-to-PDF.

server/evidence.py
🔐

Identity & channel

Per-agent keys hashed at rest, constant-time compared, rotatable & revocable; HTTPS enforced for any credential off the box.

server/auth.py · identity.py
